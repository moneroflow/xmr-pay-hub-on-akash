## Referral Sync Auto-Start Fix (April 17, 2026)

### Issue
Referral telemetry sync stopped when users refreshed page or visited `/dashboard` directly. Users saw ZERO sync API calls for hours in backend logs (`access.log`). Only clicking "Start Accepting XMR" button triggered sync again.

### Root Cause
`startReferralSync()` was only called in `login()` function, not when DashboardLayout mounts for already-authenticated users.

### Fix Applied
- Added `useEffect` in DashboardLayout to auto-start sync when `isAuthenticated` is true
- Improved error handling in referral-sync.ts (exponential backoff, visibility state handling)
- Sync now starts automatically on dashboard mount/refresh

### Privacy Note
Sync sends privacy-focused referral telemetry only (referral code, pro status, referral stats). No private keys, seeds, wallet addresses, or transaction details transmitted. Sync clearly labeled as "Privacy Focused Referrals Data Only".

## Backend Timezone Fix (April 17, 2026)

### Issue
Backend access.log showing UTC timestamps instead of AEST (Australia/Sydney) timezone.

### Root Cause
Morgan logger using default `:date[iso]` token which outputs ISO-8601 with `Z` suffix (UTC).

### Fix Applied
- Custom morgan token added to output ISO-8601 with timezone offset (+10/+11 based on DST)
- Set `process.env.TZ = 'Australia/Sydney'` in backend
- Fixed PM2 restart issue (45 restarts due to port conflicts from stale process)

### Files Changed
- `mf-backend/index-final.js` - Timezone-aware logging, PM2 restart

## PM2 Backend Recovery (April 17, 2026)

### Issue
